วิธีใช้งาน (Quick Start บน Demo)

1
เปิดหน้าหลัก → กด ทดลองใช้งาน
2
ลากไฟล์ เอกสารเข้ากล่อง upload (PDF/Word/Excel/TXT/PNG)
3
รอประมวลผล — extract → chunk → enrich → embed → wiki (ใช้เวลา 10-60 วินาที)
4
ถามคำถาม ในกล่อง chat ด้านล่าง — ระบบจะค้นจากเอกสารคุณก่อน แล้วตอบ
5
เปิด Wiki (ปุ่ม 📖 มุมขวาบน) เพื่อดูสรุป tags, key points, sections, suggested questions

วิธีติดตั้งบนเครื่องตัวเอง (Self-Host)

⚠️ สำคัญ: ZIP ที่ดาวน์โหลด ไม่มี API key ของ server (ถูกตัดออกเพื่อความปลอดภัย) → คุณต้องสมัคร Gemini API key ของตัวเอง (ฟรี) แล้วใส่ใน .env หรือผ่านหน้า Setup

1. Requirements

  • Python 3.10+ (แนะนำ 3.12)
  • RAM ขั้นต่ำ 2 GB (ถ้าใช้ OCR แนะนำ 4 GB)
  • OS: Linux / macOS / Windows (พร้อม WSL2)
  • Gemini API Key (ฟรี — ขอที่ aistudio.google.com)

2. Download & Extract

# ดาวน์โหลด zip จากปุ่มด้านบน → unzip
unzip ai-chat-demo.zip
cd ai-chat-demo

3. ติดตั้ง dependencies

python3 -m venv venv
source venv/bin/activate          # Linux/Mac
# venv\Scripts\activate          # Windows
pip install -r requirements.txt

4. ตั้งค่า API Key

cat > .env <<EOF
GEMINI_API_KEY=AIzaSy...your-key-here
GEMINI_API_KEY_2=
GEMINI_API_KEY_3=
PORT=5000
EOF
💡 BYOK: ถ้าไม่ใส่ใน .env ก็เพิ่มผ่าน UI ในหน้า Setup ได้

5. รัน server

# Dev mode
python3 server.py

# Production (Linux/Mac)
gunicorn -w 4 -b 0.0.0.0:5000 server:app

# หรือใช้ script
bash run.sh

เปิด browser ไป http://localhost:5000 🎉

Architecture สรุป

Layerเทคโนโลยี
BackendFlask + gunicorn (port 5000)
StorageSQLite (chunks) + FAISS (vectors 3072-dim)
LLMGemini 2.5 Flash + gemini-embedding-001
PipelineExtract → Chunk(800/100) → Enrich ‖ Embed → Wiki
RAGLevel 4: classify → multi-query → hybrid → AI rerank → reflect
FrontendVanilla JS (IIFE) + CDN libs (no build step)
AuthCookie-based session isolation (no login required)

Security Notes

Multi-user isolation — แต่ละ browser session เห็นเฉพาะไฟล์ตัวเอง
API keys ไม่ถูก commit — เก็บใน .env (อยู่ใน .gitignore)
Cookie HttpOnly + SameSite=Lax — ป้องกัน XSS อ่าน sid
⚠️ Production: ถ้า deploy จริง ควร reverse-proxy ผ่าน nginx + HTTPS และเพิ่ม authentication layer
